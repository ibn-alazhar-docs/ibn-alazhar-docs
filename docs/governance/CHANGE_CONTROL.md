# Change Control

## Proposal

Submit a change proposal via spec in `specs/NNN-feature/spec.md`:

- Problem statement
- Proposed solution
- Scope and boundaries
- Phase alignment
- Dependencies

## Approval

1. Spec review by spec-guardian
2. Architecture review (if architectural impact)
3. Phase gate check — does this belong in the current phase?
4. Final approval before implementation begins

## Emergency Changes

- Hotfixes bypass full process but require:
  1. Clear problem identification
  2. Minimal fix (no scope expansion)
  3. Post-fix review within 24h
  4. Spec update documenting the change
