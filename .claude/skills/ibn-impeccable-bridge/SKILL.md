# Ibn Al-Azhar Docs × Impeccable Bridge

Before using any Impeccable design command, read:

1. PRODUCT.md
2. DESIGN.md
3. CLAUDE.md
4. docs/29_BRAND_IMPLEMENTATION_GUIDE.md if present
5. docs/04_UI_DESIGN_SYSTEM.md if present

## Mandatory adaptation rules

- Arabic-first.
- RTL-first.
- Cairo typography.
- Primary Green #16A34A.
- Heritage Gold #CA8A04 as restrained accent only.
- Dark Text Gray #1F2937.
- No generic SaaS gradients.
- No random glassmorphism.
- No fake analytics.
- No English-first layouts.
- No decorative Islamic patterns unless functional and subtle.
- Every UI must include loading, empty, error, and success states where relevant.
- Every design decision must support study workflows.

## Impeccable usage guidance

Use Impeccable for:
- audit
- polish
- critique
- typeset
- arrange
- clarify
- harden
- document
- teach

But never accept Impeccable output blindly. Re-check against:
- ibn-brand-audit
- ibn-rtl-audit
- ibn-frontend-polish
- ibn-qa-check

## Output

Return:
- Impeccable command used or recommended
- Brand/RTL adaptations required
- Files affected
- Risks
- Follow-up checks
