# Spec Authority

## Lifecycle

1. **Draft** — Initial proposal, under discussion
2. **Review** — Under review by spec-guardian and stakeholders
3. **Approved** — Passed gate, ready for implementation
4. **Implemented** — Code merged, tests passing
5. **Archived** — Superseded or completed

## Requirements

- Must reference the target phase
- Must define scope, non-goals, and success criteria
- Must include acceptance criteria
- Must identify dependencies
- Must be reviewed before implementation begins

## Authority

- Approved specs override ambiguous requirements
- Specs take precedence over code comments
- Spec changes require version bump and re-review
