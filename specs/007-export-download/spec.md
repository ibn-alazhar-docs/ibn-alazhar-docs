# Spec 007: Enhanced Export & Download

## Status

- **Status**: approved
- **Phase**: 2C-3
- **Created**: 2026-06-17
- **Author**: Antigravity

## Overview

يتيح التصدير المتقدم للمستخدمين تحويل المستندات المعالجة (Markdown/Text) إلى تنسيقات احترافية متعددة تلبي احتياجات الطلاب الأزهريين، مع دعم كامل للغة العربية (RTL) وخيارات تنسيق متقدمة، بالإضافة إلى إمكانية الحفظ المباشر في Google Drive.

## User Stories

1. **كطالب أزهري**، أريد تصدير مستنداتي بصيغة **Word (.docx)** لأتمكن من تعديل الأبحاث وإضافة الحواشي بسهولة.
2. **كمعلم**، أريد تصدير المذكرات بصيغة **PDF** بتنسيق ممتاز واتجاه RTL صحيح ليتم طباعتها مباشرة للطلاب.
3. **كمستخدم متنقل**، أريد حفظ المستندات المُصدرة **مباشرة إلى حسابي في Google Drive** لأتمكن من الوصول إليها من هاتفي المحمول.
4. **كباحث**، أريد التحكم في **خيارات التصدير** (مثل تحديد حجم الخط، إضافة علامة مائية، وتحديد صفحات معينة) لتخصيص شكل المستند النهائي.

## Acceptance Criteria

1. **تصدير PDF**:
   - إنشاء ملفات PDF تدعم اللغة العربية تماماً (Cairo/Amiri Fonts).
   - الحفاظ على الاتجاه من اليمين لليسار (RTL).
   - دعم التنسيقات (Bold, Italic, Tables, Lists).
2. **تصدير Word (DOCX)**:
   - إنشاء ملفات `.docx` قابلة للتحرير بالكامل.
   - دعم اتجاه RTL وتنسيقات النصوص العربية.
3. **التصدير إلى Google Drive**:
   - زر "حفظ في Google Drive" يرسل الملف المصدّر مباشرة إلى سحابة المستخدم (بناءً على التكامل الحالي في الـ Pipeline).
4. **الخيارات المتقدمة (Export Modal)**:
   - نافذة منبثقة (Modal) تظهر عند النقر على "تصدير".
   - تتيح اختيار الصيغة (PDF, DOCX, TXT, MD).
   - تتيح إدخال خيارات إضافية: حجم الخط، تحديد نطاق الصفحات (إن أمكن برمجياً)، وإضافة علامة مائية بسيطة (Watermark).

## UI/UX Requirements

- إضافة مكون للتحكم بالتصدير المتقدم (Export Options) في صفحة معاينة المستند `preview/[id]/page.tsx` وربما في جدول الملفات.
- تصميم **Export Modal** متوافق مع تصميم `shadcn/ui` والهوية البصرية للمشروع.
- توفير حالات تحميل (Loading States) عند طلب التصدير المخصص، حيث يتم إرسال طلب للـ Queue والانتظار حتى انتهاء المهمة وتوفر رابط التحميل أو إشعار نجاح الرفع لـ Drive.

## API Endpoints

- `POST /api/documents/:id/export`
  - Body: `{ format: 'pdf' | 'docx' | 'md' | 'txt', options?: { fontSize?: number, watermark?: string, pageRange?: string, destination?: 'download' | 'gdrive' } }`
  - يرسل وظيفة للـ `export-worker` عبر BullMQ.
  - يعود بـ `jobId` ليتم تتبعه.
- `GET /api/documents/:id/export/:format/status` لتتبع حالة التصدير.

## Pipeline & Worker Changes

- تحديث `export-worker` ليدعم صيغة `pdf` باستخدام مكتبة مثل `pdfmake` أو `puppeteer` مع دعم خطوط عربية، أو تحسين `docx` لتصدير PDF.
- تحديث واجهة `ExportRequest` لتشمل خيارات `options` (حجم الخط، العلامة المائية).
- تطبيق العلامة المائية وحجم الخط في معالجة الـ PDF والـ Word قبل تصديرها.
