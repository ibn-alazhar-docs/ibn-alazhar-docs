## 1. منهجية تقييم المخاطر

### مقياس الاحتمالية

| المستوى | القيمة | الوصف |
|---|---|---|
| ضعيف جداً (Very Low) | 1 | حدث نادر جداً، غير متوقع |
| ضعيف (Low) | 2 | قد يحدث لكن بشكل غير محتمل |
| متوسط (Medium) | 3 | قد يحدث بشكل معقول |
| عالي (High) | 4 | مرجح الحدوث |
| عالي جداً (Very High) | 5 | شبه مؤكد الحدوث |

### مقياس التأثير

| المستوى | القيمة | الوصف |
|---|---|---|
| ضعيف جداً (Very Low) | 1 | تأثير طفيف، يمكن تجاوزه |
| ضعيف (Low) | 2 | تأثير محدود، يحتاج تعديل بسيط |
| متوسط (Medium) | 3 | تأثير ملحوظ، يؤخر المشروع أو يقلل الجودة |
| عالي (High) | 4 | تأثير كبير، قد يوقف ميزة أو مرحلة |
| عالي جداً (Very High) | 5 | تأثير كارثي، يهدد المشروع بالكامل |

### درجة المخاطر = الاحتمالية × التأثير

| الدرجة | التصنيف | اللون |
|---|---|---|
| 1–4 | منخفض (Low) | 🟢 |
| 5–9 | متوسط (Medium) | 🟡 |
| 10–15 | عالي (High) | 🟠 |
| 16–25 | حرج (Critical) | 🔴 |

### RISK-001: استنفاد حصة Google Drive API

| الحقل | القيمة |
|---|---|
| **معرف المخاطرة** | RISK-001 |
| **التصنيف** | تقني (Technical) |
| **الوصف** | Google Drive API يفرض حصة يومية على طلبات OCR. عند تجاوز الحصة، يتوقف تحويل المستندات بالكامل حتى اليوم التالي. الحصة الافتراضية محدودة وقد لا تكفي مع زيادة المستخدمين. |
| **الاحتمالية** | عالي (4) |
| **التأثير** | عالي (4) |
| **درجة المخاطرة** | 16 — 🔴 حرج |
| **استراتيجية التخفيف** | 1. مراقبة الاستهلاك اليومي عبر لوحة تحكم المدير 2. تطبيق rate limiting لكل مستخدم (مثلاً: 10 تحويلات/يوم) 3. طلب زيادة الحصة من Google Cloud Console 4. إضافة نظام طابور مع أولويات |
| **خطة الطوارئ** | 1. عرض رسالة واضحة للمستخدم: "تم الوصول للحد اليومي، حاول غداً" 2. تفعيل وضع القراءة فقط للملفات الموجودة 3. دراسة بديل OCR محلي (Tesseract) كـ fallback |

| **تاريخ المراجعة** | نهاية Phase 3 |
