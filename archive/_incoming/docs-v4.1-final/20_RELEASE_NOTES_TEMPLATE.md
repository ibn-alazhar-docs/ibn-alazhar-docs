### الملخّص

نُعلن عن إطلاق الإصدار الأول من منصة **Ibn Al-Azhar Docs — ابن الأزهر دوكس** — منصة الويب التقدّمية (PWA) مفتوحة المصدر لإدارة المستندات التعليمية وتحويلها. هذا الإصدار يتضمن خط أنابيب التحويل الكامل: رفع الملفات، تقسيم PDF، التعرف البصري على الحروف عبر Google Drive OCR (التحويل/Conversion)، والتصدير (Export) بصيغ TXT وDOCX وJSON. المنصة تدعم العربية بشكل أصيل مع واجهة RTL كاملة، وتعمل كتطبيق PWA قابل للتثبيت على أي جهاز.

---

### 🆕 ميزات جديدة

#### رفع الملفات وإدارتها

**الوصف:** رفع ملفات PDF والصور (PNG, JPEG, WEBP) بحجم أقصى 100MB مع تخزين آمن في MinIO (حاوية `ibn-al-azhar-docs-files`). نظام ملفات هرمي يدعم المجلدات والسحب والإفلات والتحديد المتعدد وسلة المحذوفات.

**لقطة شاشة:**

![واجهة رفع الملفات](./screenshots/v0.1.0/file-upload.png)

**المرجع:** #12, #15, #18

---

#### خط أنابيب التحويل (Conversion) عبر OCR

**الوصف:** تحويل المستندات الممسوحة ضوئيًا إلى نصوص رقمية عبر خط أنابيب متكامل: تقسيم PDF إلى صفحات → OCR عبر Google Drive API → تنظيف النص → تجميع النتائج. تتبّع التقدّم في الوقت الفعلي عبر SSE مع إشعارات فورية. **ملاحظة:** التحويل (Conversion) يختلف عن التصدير (Export) — التحويل يستخرج النص فقط، والتصدير يُولّد الملف بصيغة معينة.

**لقطة شاشة:**

![تقدّم التحويل](./screenshots/v0.1.0/conversion-progress.png)

**المرجع:** #22, #25, #28

---

#### التصدير (Export) بصيغ متعددة

**الوصف:** تصدير نتائج OCR المُستخرَجة مسبقاً بصيغ TXT (نص عادي) وDOCX (مستند Word مع دعم RTL) وJSON (بيانات مهيكلة بترتيب الصفحات). تحميل مباشر عبر Presigned URLs من MinIO. **ملاحظة:** التصدير مرحلة منفصلة عن التحويل — يمكن إعادة التصدير بصيغة مختلفة بدون إعادة OCR.

**لقطة شاشة:**

![خيارات التصدير](./screenshots/v0.1.0/export-options.png)

**المرجع:** #35, #37

---

#### المصادقة وإدارة الحسابات

**الوصف:** تسجيل الدخول عبر البريد الإلكتروني وكلمة المرور أو Google OAuth. ثلاثة أدوار: طالب، معلم، ومدير مع صلاحيات مختلفة. جلسات JWT (NextAuth.js v5) مدتها 24 ساعة — بدون access/refresh token منفصل. أول حساب مدير يُنشأ عبر seed script مع متغيرات البيئة.

**لقطة شاشة:**

![صفحة تسجيل الدخول](./screenshots/v0.1.0/login-page.png)

**المرجع:** #8, #10

---

#### واجهة عربية RTL أصيلة

**الوصف:** واجهة مستخدم كاملة بتصميم RTL-first مع دعم ثنائي الاتجاه (عربي/إنجليزي) عبر next-intl. تبديل اللغة سلس مع حفظ التفضيل. جميع المكونات تدعم الاتجاهين.

**لقطة شاشة:**

![الواجهة العربية](./screenshots/v0.1.0/arabic-ui.png)

**المرجع:** #5, #7

---

#### تثبيت PWA والعمل دون اتصال

**الوصف:** تثبيت التطبيق كتطبيق مستقل على سطح المكتب والهاتف المحمول. صفحة App Shell مُخزّنة مسبقًا للتحميل الفوري. استراتيجية Network First لطلبات API مع fallback للـ cache. صفحة offline مخصصة.

**لقطة شاشة:**

![موجه التثبيت](./screenshots/v0.1.0/install-prompt.png)

**المرجع:** #42, #44

---

#### المشاركة عبر روابط عامة

**الوصف:** مشاركة الملفات والمجلدات عبر روابط عامة تتيح العرض والتحميل دون تسجيل دخول. روابط قابلة للإلغاء في أي وقت. الرمز (token) يتكون من 64 حرف hex (32 بايت عشوائية).

**المرجع:** #50

---

#### لوحة الإدارة

**الوصف:** لوحة تحكم للمديرين تتضمن إحصائيات الاستخدام (عدد المستخدمين، التحويلات، التخزين) وإدارة المستخدمين (تعليق، تفعيل، تغيير الدور).

**المرجع:** #55, #57

### تعليمات الترقية

#### تثبيت جديد

```bash
# 1. استنساخ المستودع
git clone https://github.com/ibn-al-azhar-docs/ibn-al-azhar-docs.git
cd ibn-al-azhar-docs
git checkout v0.1.0

# 2. تثبيت التبعيات
pnpm install

# 3. نسخ ملف المتغيرات البيئية
cp .env.example .env

# 4. تحرير المتغيرات البيئية
# أهم المتغيرات:
#   AUTH_SECRET — توليد: openssl rand -hex 32
#   ADMIN_EMAIL — بريد المدير الأول
#   ADMIN_PASSWORD — كلمة مرور المدير الأول
#   GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET — من Google Cloud Console
#   DATABASE_URL — اتصال PostgreSQL
$EDITOR .env

# 5. تشغيل الخدمات الأساسية
docker compose up -d db redis minio minio-init

# 6. تطبيق ترحيلات قاعدة البيانات
pnpm prisma migrate deploy

# 7. بذر البيانات الأولية (ينشئ حساب المدير من ADMIN_EMAIL/ADMIN_PASSWORD)
pnpm prisma db seed

# 8. بناء التطبيق
pnpm build

# 9. تشغيل خادم التطوير
pnpm dev
```

#### نشر عبر Docker (الإنتاج)

```bash
# 1. على الخادم
cd /opt/ibn-al-azhar-docs
git clone https://github.com/ibn-al-azhar-docs/ibn-al-azhar-docs.git .
git checkout v0.1.0

# 2. إعداد المتغيرات البيئية
cp .env.example .env
nano .env  # أدخل قيم الإنتاج

# 3. تشغيل جميع الخدمات
docker compose -f docker-compose.prod.yml up -d

# 4. التحقق
curl -f http://localhost:3000/api/health
```
