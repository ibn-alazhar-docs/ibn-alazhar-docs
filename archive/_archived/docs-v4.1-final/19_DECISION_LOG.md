## مقدمة

يُوثّق هذا السجل جميع القرارات الرئيسية المُتّخَذة خلال بناء منصة **Ibn Al-Azhar Docs — ابن الأزهر دوكس**، بدءًا من الاختيارات التقنية الكبرى وصولًا إلى قرارات التسمية والتنظيم. يختلف هذا السجل عن سجلّ قرارات العمارة (ADR) بأنه يشمل جميع أنواع القرارات—not just architectural—بما في ذلك قرارات المنتج والعمليات والمجتمع.

### الهدف

- توفير سياق تاريخي لكل قرار رئيسي
- تسهيل على أعضاء الفريق الجدد فهم "لماذا" خلف كل اختيار
- منع إعادة فتح النقاشات المُفضَلة دون معلومات جديدة جوهرية
- دعم عملية المراجعة والتدقيق

### مصطلحات الحالة

| الحالة | المعنى |
|--------|--------|
| **مقبول** | القرار مُتّخَذ ونافذ |
| **مُعلّق** | القرار قيد المناقشة أو بانتظار معلومات إضافية |
| **مُستبدَل** | القرار أُلغي واستُبدِل بقرار آخر |
| **مُهمَل** | القرار لم يعد ذا صلة |

## دورة حياة القرار

```
اقتراح قرار
    │
    ▼
مناقشة الفريق ← ─ ─ ─ ─ ─ ─ ─
    │                            │
    ▼                            │
جمع المعلومات والبدائل          │
    │                            │
    ▼                            │
تقييم المقايضات                 │
    │                            │
    ▼                            │ (إذا ظهرت معلومات جديدة)
اتخاذ القرار                    │
    │                            │
    ▼                            │
توثيق القرار (هذا السجل)  ─ ─ ─
    │
    ▼
تنفيذ القرار
    │
    ▼
مراجعة لاحقة (اختياري)
```

### قواعد اتخاذ القرار

1. **مبدأ الاكتفاء:** القرار يُتّخذ عندما تكون المعلومات المتاحة كافية—لا ننتظر اليقين التام
2. **مبدأ القابلية للعكس:** القرارات القابلة للعكس (نوع المكتبة) تُتّخذ بسرعة أكبر من القرارات غير القابلة للعكس (نوع قاعدة البيانات)
3. **مبدأ التأثير:** القرارات ذات التأثير العالي تتطلب مناقشة أوسع وموافقة أكثر
4. **مبدأ التوثيق:** كل قرار يُوثّق مع السياق والمبرّر—لا يُكتفي بـ "اتفقنا على..."

### تصنيف القرارات حسب التأثير

| التصنيف | الوصف | أمثلة | مَن يُقرّ |
|---------|--------|-------|----------|
| **حرج** | يؤثر على البنية الكلية أو الأمان أو البيانات | نوع المنصة، قاعدة البيانات، المصادقة | الفريق كامل + مراجع خارجي |
| **عالٍ** | يؤثر على عدة مكوّنات أو تجربة المستخدم | إطار العمل، مكتبة الواجهة، OCR | الفريق التقني |
| **متوسط** | يؤثر على مكوّن واحد أو سير العمل | ORM، طابور المهام، اختبار | المطوّر المسؤول |
| **منخفض** | يؤثر على التفضيلات الشخصية أو التنظيم | مكتبة التسجيل، تسمية الملفات | أي مطوّر مع توثيق |
