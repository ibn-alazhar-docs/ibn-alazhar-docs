## A

| المصطلح الإنجليزي | المصطلح العربي | التعريف |
|---|---|---|
| **Accessibility** | إمكانية الوصول | ممارسات تصميم وتطوير تضمن إمكانية استخدام التطبيق من قبل جميع الأشخاص، بمن فيهم ذوو الإعاقة. تشمل دعم قارئات الشاشة والتنقل بلوحة المفاتيح والتباين الكافي. معيار WCAG 2.1 Level AA هو المستهدف. |
| **API (Application Programming Interface)** | واجهة برمجة التطبيقات | مجموعة من القواعد والبروتوكولات التي تسمح لتطبيقات مختلفة بالتواصل مع بعضها. في ابن الأزهر دوكس، Next.js API routes توفر endpoints للتفاعل مع الخادم. |
| **API Route** | مسار API | نقطة نهاية (endpoint) في Next.js تُعالج طلبات HTTP. تقع في مجلد `app/api/` وتُرجع بيانات JSON. |
| **App Router** | موجه التطبيق | نظام التوجيه الجديد في Next.js 13+ يعتمد على مجلدات الملفات داخل `app/`. يدعم Server Components وlayouts وتحميل تدريجي. |
| **ARIA (Accessible Rich Internet Applications)** | تطبيقات الإنترنت الغنية القابلة للوصول | مجموعة من السمات HTML التي تُحسّن إمكانية الوصول للتطبيقات الديناميكية. أمثلة: `aria-label`, `aria-hidden`, `role`. |
| **Auth (Authentication)** | المصادقة / التوثيق | عملية التحقق من هوية المستخدم. في ابن الأزهر دوكس، NextAuth.js v5 يُدير المصادقة عبر JWT strategy (24h) مع البريد الإلكتروني وكلمة المرور. لا يُستخدم access/refresh token منفصل. |
| **Auto-TLS** | تشفير تلقائي | ميزة في Caddy تُصدر وتُجدّد شهادات SSL/TLS تلقائياً عبر Let's Encrypt بدون تدخل يدوي. |

## B

| المصطلح الإنجليزي | المصطلح العربي | التعريف |
|---|---|---|
| **Backlog** | سجل المهام المؤجلة | قائمة المهام المرتبة حسب الأولوية التي لم تُنفَّذ بعد. تُستخدم في إدارة المشاريع الرشيقة (Agile). |
| **BCrypt** | خوارزمية تشفير BCrypt | خوارزمية تشفير أحادية الاتجاه (hashing) مُصمّمة لكلمات المرور. بطيئة عمداً لمنع هجمات القوة الغاشمة. تُستخدم في ابن الأزهر دوكس لتخزين كلمات المرور بأمان. |
| **BullMQ** | نظام طوابير BullMQ | مكتبة Node.js لإدارة طوابير المهام (job queues) مبنية على Redis. تُستخدم في ابن الأزهر دوكس لمعالجة OCR بشكل غير متزامن: رفع الملف ← إضافة مهمة ← معالجة Worker ← إرجاع النتيجة. |
| **Bundle Size** | حجم الحزمة | حجم ملفات JavaScript المُجمّعة التي يُحمّلها المتصفح. الحجم الأصغر يعني تحميل أسرع. المستهدف في ابن الأزهر دوكس: < 200KB مضغوط. |

## C

| المصطلح الإنجليزي | المصطلح العربي | التعريف |
|---|---|---|
| **Caddy** | خادم Caddy | خادم ويب (reverse proxy) يتميز بـ auto-TLS وتهيئة بسيطة. القرار النهائي: Caddy هو الـ reverse proxy المُختار لابن الأزهر دوكس بدلاً من Nginx لسهولة إدارة شهادات SSL. راجع [19_DECISION_LOG.md](./19_DECISION_LOG.md) D-2025-03-05-V4-08. |
| **CDN (Content Delivery Network)** | شبكة توصيل المحتوى | شبكة خوادم موزعة جغرافياً تُخزّن نسخاً من المحتوى وتقدّمه من أقرب خادم للمستخدم. يُسرّع تحميل الملفات الثابتة. |
| **CI/CD (Continuous Integration/Continuous Deployment)** | التكامل المستمر / النشر المستمر | ممارسات أتمتة اختبار ونشر الكود. في ابن الأزهر دوكس: GitHub Actions يُشغّل lint → test → build → deploy تلقائياً عند كل push. |
| **Connection Pool** | تجمع الاتصالات | مجموعة من اتصالات قاعدة البيانات المُعاد استخدامها لتجنب تكلفة إنشاء اتصال جديد لكل طلب. Prisma يُدير تجمع الاتصالات تلقائياً. |
| **Conversion** | تحويل (استخراج) | عملية استخراج النص من المستندات الممسوحة أو الصور عبر OCR. Conversion ≠ Export: التحويل هو استخراج النص فقط، بينما التصدير هو توليد ملف بصيغة معينة. راجع [19_DECISION_LOG.md](./19_DECISION_LOG.md) D-2025-03-05-V4-05. |
| **CORS (Cross-Origin Resource Sharing)** | مشاركة الموارد عبر الأصول | آلية أمان في المتصفح تتحكم في أي domains يمكنها الوصول لـ API. في ابن الأزهر دوكس: Same-origin + specific allowed origins في الإنتاج. راجع [19_DECISION_LOG.md](./19_DECISION_LOG.md) D-2025-03-05-V4-12. |
| **CSP (Content Security Policy)** | سياسة أمان المحتوى | header أمان يحدد مصادر المحتوى المسموحة (سكربتات، صور، أنماط). يمنع هجمات XSS عبر منع تحميل سكربتات من مصادر غير موثوقة. |
| **CSRF (Cross-Site Request Forgery)** | تزوير الطلبات عبر المواقع | هجوم يُجبر متصفح المستخدم على إرسال طلبات غير مصرح بها للتطبيق. NextAuth.js v5 يوفر حماية CSRF تلقائية عبر tokens. |
| **CSS (Cascading Style Sheets)** | أوراق الأنماط المتتالية | لغة تنسيق تتحكم في مظهر صفحات الويب. في ابن الأزهر دوكس: Tailwind CSS v4 يُستخدم لجميع الأنماط. |

## D

| المصطلح الإنجليزي | المصطلح العربي | التعريف |
|---|---|---|
| **Dependency** | تبعية / مكتبة طرف ثالث | مكتبة أو حزمة خارجية يعتمد عليها المشروع. مثال: `pdfjs-dist` هي تبعية لعرض PDF. إدارة التبعيات تتم عبر pnpm. |
| **Docker** | نظام حاويات Docker | منصة لحزم التطبيقات في حاويات (containers) معزولة تشمل كل ما تحتاجه للعمل. في ابن الأزهر دوكس: جميع الخدمات تعمل في حاويات Docker. |
| **Docker Compose** | منسق حاويات Docker Compose | أداة لتحديد وتشغيل تطبيقات متعددة الحاويات. في ابن الأزهر دوكس: `docker-compose.yml` يُعرّف خدمات web, worker, postgres, redis, minio, caddy. |
| **DOM (Document Object Model)** | نموذج كائن المستند | تمثيل برمجي لصفحة HTML يسمح لـ JavaScript بالتعامل مع عناصر الصفحة ديناميكياً. React يُحدّث DOM افتراضياً (Virtual DOM). |
| **Drag & Drop** | السحب والإفلات | نمط تفاعل يسمح للمستخدم بسحب ملفات من سطح المكتب وإفلاتها في منطقة الرفع. مدعوم في ابن الأزهر دوكس لرفع الملفات. |

## E

| المصطلح الإنجليزي | المصطلح العربي | التعريف |
|---|---|---|
| **E2E Testing (End-to-End)** | اختبار شامل | اختبار يحاكي سلوك المستخدم الحقيقي من البداية للنهاية عبر المتصفح. في ابن الأزهر دوكس: Playwright يُستخدم لاختبار مسارات مثل رفع ملف ← تحويل (OCR) ← تصدير. |
| **Endpoint** | نقطة نهاية | عنوان URL محدد في API يستقبل طلبات HTTP ويرجع استجابات. مثال: `POST /api/files/upload` نقطة نهاية لرفع الملفات. |
| **Environment Variable** | متغير بيئة | قيمة إعداد تُعرّف خارج الكود (في `.env` أو النظام) لفصل الإعدادات الحساسة عن الكود المصدري. مثل: `DATABASE_URL`, `MINIO_ACCESS_KEY`, `ADMIN_EMAIL`, `ADMIN_PASSWORD`. |
| **Exponential Backoff** | تراجع أسي | استراتيجية لإعادة المحاولة حيث يزداد وقت الانتظار بشكل أسي بعد كل فشل. مثال: 1s → 2s → 4s → 8s. تُستخدم في BullMQ لإعادة محاولة المهام الفاشلة. |
| **Export** | تصدير | عملية توليد ملف بصيغة قابلة للتنزيل من النص المُستخرَج مسبقاً. التصدير يحدث **بعد** التحويل (Conversion) وليس جزءاً منه. في ابن الأزهر دوكس: تصدير نتائج OCR بصيغ TXT, DOCX, أو JSON. راجع [19_DECISION_LOG.md](./19_DECISION_LOG.md) D-2025-03-05-V4-05. |

## التفريق بين Conversion وExport

> **ملاحظة مهمة:** من القرارات الأساسية في V4 (D-2025-03-05-V4-05) أن التحويل (Conversion) والتصدير (Export) مفهومان منفصلان:

| المفهوم | التعريف | المخرجات | مثال |
|---------|---------|----------|------|
| **Conversion (تحويل)** | استخراج النص من المستندات الممسوحة/الصور عبر OCR | نص مُستخرَج يُخزَّن في قاعدة البيانات | ملف PDF ممسوح ← نص رقمي قابل للبحث |
| **Export (تصدير)** | توليد ملف بصيغة معينة من النص المُستخرَج مسبقاً | ملف قابل للتنزيل (TXT, DOCX, JSON) | نص مُستخرَج ← ملف DOCX مع تنسيق RTL |

**التسلسل:** رفع ملف → **Conversion** (استخراج OCR) → تخزين النص → **Export** (توليد ملف بصيغة)
